What You Can Do

Ask Questions

  • Forum
  • #jquery on IRC

Answer Questions

  • Get a little, give a little
  • Explaining things to others solidifies your own learning and helps spread the love

Publish Your Discoveries

  • Write about what you've learned
    • Blog it
    • Comment on other blogs

Share your plugins

  • Keep the code on a public code repository
    • GitHub
    • Google Code
  • Upload to the official jQuery plugin repository
  • Write clear, comprehensive documentation
    • Example: Roundabout Plugin
    • Example: Form Plugin

Report Bugs

  • Use the jQuery bug tracker (currently at http://dev.jquery.com/newticket/)
  • Post a description of the bug to the appropriate dev list (currently at http://groups.google.com/group/jquery-dev/)
  • Provide a reduced test case

    • Fork jQuery

    • For experienced JavaScript developers only
    • Learn Git
    • Fork the jQuery repo on GitHub
    • Send a pull request
    • Use jQuery's test suite to ensure that your changes did not break anything.
    • Add your own tests to cover changes / additions to the code.

    Pace Yourself

    • Plugin management
      • Good documentation is key
      • If the plugin isn't ready for release or you don't have time to support it, don't put it on plugins.jquery.com.
      • You can still have it publicly available on a version-controlled code repository.

    Pace Yourself

    • Blog comment management
      • Comments are an important way to learn and improve.
      • But comments can get overwhelming.
      • Replying to comments can often take longer, in total, than writing the blog entry.
      • Before you post a blog entry, make sure you'll have at least a couple hours within the following few days to respond to comments
